Головний опис роботи

Бакалаврська робота присвячена створенню веборієнтованої інформаційної системи дистанційного навчання з інтеграцією особистого кабінету. Основний акцент зроблено на надійному backend-рішенні, яке підтримує гнучке керування курсами, користувачами та освітнім контентом.

Актуальність теми

Цифровізація освіти збільшує попит на стабільні та масштабовані платформи, здатні працювати з великою кількістю студентів і викладачів одночасно. Університетам потрібні рішення, що поєднують централізоване адміністрування, швидкий доступ до навчальних матеріалів та прозорий контроль прогресу навчання.

Запропонована система відповідає цим викликам завдяки модульній архітектурі, стандартизованому API та продуманій моделі ролей, що знижує технічні ризики під час подальшого впровадження.

Мета та завдання дослідження

Мета

Розробити backend-ядро системи дистанційного навчання, яке забезпечує безпечну взаємодію ролей, керування освітнім контентом та готовність до промислового масштабування.

Основні завдання

  • Проаналізувати предметну область та сформувати ключові функціональні вимоги.
  • Спроєктувати багаторівневу архітектуру із чітким розподілом відповідальностей.
  • Реалізувати API для роботи з користувачами, курсами, уроками та ролями.
  • Інтегрувати логування, обробку помилок і механізми розширення системи.
  • Підготувати основу для подальшої інтеграції з веб- і мобільними клієнтами.

Методологія дослідження

Дослідження виконано ітеративно: від аналізу вимог і архітектурного проєктування до реалізації, тестування та оцінювання результатів. Для зменшення складності використано шаблони CQRS-підходу з MediatR, а також практики чистої архітектури для довгострокової підтримки коду.

Архітектура рішення

Серверна частина

У роботі застосовано N-Layer підхід та MediatR для чіткого розподілу відповідальностей між командами, запитами та інфраструктурними компонентами.

Технологічний стек

  • ASP.NET Core
  • Entity Framework Core
  • MS SQL Server
  • MediatR
  • Serilog

Очікувані результати

Практична цінність

Результатом є модульний та масштабований backend-прототип освітньої платформи, готовий до інтеграції з frontend-рішеннями та подальшого розширення.

  • Зменшення часу відповіді основних API-запитів та покращення стабільності сервісу.
  • Підвищення керованості навчального контенту для викладачів та адміністраторів.
  • Підготовлена основа для підключення аналітики прогресу студентів.
  • Технічна готовність до подальшого масштабування інфраструктури.
